# Future backlog ideas not yet assigned to a phase

Ideas captured from the original feature specification brief. Not yet assigned to a specific phase.

## Frustration Signal Detection

Ostii intentionally swears during sessions to mark frustration. An agent could scan session transcripts for these linguistic markers and surface patterns about recurring failure points. Partially addressed in Phase 1 via prompt-level semantic signal detection, but a deeper transcript-scanning approach could be a standalone feature.

## Manual Entry Support

Allow user to manually add an observation or note to a session outside of the automatic hook-driven capture. Useful for recording decisions made outside of the coding session (meetings, Slack conversations, etc.).

## Cross-Project Search

Global search across all snapshots from all projects. Currently each project's memory is siloed in the viewer. A global search would surface patterns and decisions that span multiple codebases.

## Session Sidebar Search/Filter

When the project list grows large (20-30+ projects), add search or tag filtering to the sidebar. Could include tagging projects by category, client, or tech stack.
